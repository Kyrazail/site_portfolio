    Tâche 2 : Installation et configuration d'un firewall Stormshield

    Le but de cette tâche était de configurer un firewall Stormshield. Une firewall est un équipement réseau permettant de sécuriser un réseau informatique en agissant comme un point de contrôle obligatoire entre le réseau interne et le réseau externe. Par ce positionnement dans le réseau, il peut ainsi mettre en oeuvre différents mécanismes permettant de sécuriser le réseau interne comme le filtrage, NAT, la gestion des certificats, le controle d'accès, la détection et la prévention d'intrusion, etc...
    Mon binôme et moi avons mis en place une politique de filtrage et de NAT. Le NAT est un mécanisme qui permet de remplacer une adresse IP interne par l'adresse IP publique du pare-feu ou du routeur. Nous avons également effectué une translation au niveau des ports, appelée PAT. Concernant la politique de filtrage mise en place, nous avons interdit tout accès au réseau interne depuis l'extérieur et avons seulement autorisé les connexions entrantes sur les ports web (HTTP puis HTTPS) et le port FTP (21).

    Pour en savoir plus sur cette tâche, n'hésitez pas à jeter un œil à notre rapport : cr_sae401_tache_2.pdf

    Tâche 4 : Authentification transparente par certificat SSL

    L'objectif de cette tâche était de configurer une authentification transparente par certificat SSL. Cette méthode repose sur l'utilisation de certificats numériques pour authentifier les utilisateurs de manière automatique, sans nécessiter la saisie de mot de passe. Le certificat est alors installé sur le PC de l'utilisateur et il est validé par l'instance qui a émis le certificat, donnant alors une autorisation d'accéder aux ressources du réseau. Nous avons donc configuré une autorité de certification racine (CA root) ainsi qu'un annuaire LDAP. Nous avons également mis à jour la politique de filtrage afin de définir une règle d'authentification réseau pour que les utilisateurs non authentifiés soient redirigés vers un portail captif.

    Pour en savoir plus sur cette tâche, n'hésitez pas à jeter un œil à notre rapport : cr_sae401_tache_4.pdf

    Tâche 7 : Utilisation de scanners de vulnérabilités

    La troisième tâche consistait à analyser les vulnérabilités du système d'information. J'ai effectué cette tâche seul. Une vulnérabilité dans un système d'information est une faille permettant à un attaquant de l'exploiter dans un but malveillant. Je devais réaliser un scan depuis le réseau interne puis depuis le réseau externe afin de comparer les différences entre les scans. À cette fin, j'ai utilisé différents scanners, chacun spécialisé dans la détection d'un type de faille spécifique. Les scanners que j'ai utilisés sont les suivants :

  • Nikto : Recherche de faille web
  • Nuclei : Recherche de faille web mais également les failles de l'hôte
  • Codename SCNR : Scan détaillé de l'hôte ciblé
  • Legion : Scan les services et les ports des hôtes

    • Une fois les scans effectués, j'ai pu comparer les résultats entre ceux réalisés depuis le réseau externe et ceux effectués depuis le réseau interne. J'ai ainsi pu émettre les raisons pour lesquelles les scans externes ont fourni moins d'informations. Lors de la soutenance, j'ai présenté une faille que j'ai découvert via le scanner Nuclei, nommée Ghostcat (CVE-2020-1938). Le but de cette faille est d'usurper l'identité d'un serveur Apache tomcat afin de récupérer des fichiers de configurations du vrai serveur Apache tomcat ou de créer une porte dérobée en injectant du code dans les fichiers de configurations récupérés via l'AJP (Apache Jserv Protocol : protocole de communication du serveur Apache Tomcat)
